# Stock Portfolio Optimization with Policy Iteration, Tabular Q-Learning, and Deep Q-Learning

This project simulates a simplified stock market with **N stocks**, each switching between two hidden regimes (**Low** / **High**) via a 2-state Markov process. At each time step (“day”), an agent chooses **which single stock to hold** (one action among N). The goal is to maximize long-run cumulative profit while paying a **transaction fee** when switching stocks.

The code compares three approaches:

1. **Policy Iteration (model-based dynamic programming)** — uses full transition model and expected rewards.
2. **Tabular Q-Learning (model-free RL)** — learns a Q-table by interaction.
3. **Deep Q-Learning (DQN)** — approximates Q-values with a neural network and replay memory.

---

- There are `N` stocks.
- Each stock is in one of two regimes:
  - `0 = Low`
  - `1 = High`
- The regime evolves according to fixed transition probabilities per stock.
- Each day, the agent picks **one** stock to invest in.
- The chosen stock yields a reward (profit change), sampled around its expected mean.
- If the agent switches from the previously-held stock to a new stock, a fixed **transaction fee** is subtracted.

Finally, the script runs a test simulation for `T` days and plots:
- Individual cumulative profits of each stock
- Cumulative profits achieved by each algorithm

---

## Environment / Market Model

### Regime Transition Probabilities

For each stock `i`, we define:

- `Pr{H -> H} = a`
- `Pr{H -> L} = 1 - a`
- `Pr{L -> H} = b`
- `Pr{L -> L} = 1 - b`

These are returned by:

- `create_stock_probabilities(N)`

In the current code, `a = 0.4` and `b = 0.1` for all stocks, but this can be randomized or customized.

### Reward Model

Each stock `i` has two mean rewards:

- `mu[0, i]` for Low regime
- `mu[1, i]` for High regime

Generated by:

- `create_means(N)`

Actual realized reward is sampled with small noise:

- `sample_reward(mean)` returns `Uniform(mean - 0.01, mean + 0.01)`

### Transaction Fee

If you switch from stock `old_stock` to `new_stock`, reward is reduced by:

- `transaction_fee` (called `fee` in the script)

## Algorithms Implemented
### 1) Policy Iteration

Policy Iteration assumes you know the environment model (transition probabilities + mean rewards).

Key functions:

- `create_state_space_PI(N)`
  - State representation: `(current_stock, phase_1, ..., phase_N)`
  - Total states = `N * 2^N`

- `create_state_dict_PI(N, pr, mu, transaction_fee)`
  - Builds a full Markov Decision Process (MDP) transition dictionary:
    - For each `(state, action)` enumerates all `2^N` next regime combinations with their probability and reward.

- `policy_iteration(P, gamma, epsilon)`
  - Alternates:
    - policy evaluation (compute `V`)
    - policy improvement (compute new greedy policy)

Output:
- `pi_star_PI`: near-optimal policy under the model assumptions

---

### 2) Tabular Q-Learning

Tabular Q-learning learns an action-value table `Q[s, a]` from simulated interaction.

Key functions:

- `create_state_space_Q(N)`
  - Produces a mapping from state tuples to indices.

- `play_step_Q(...)`
  - Samples the next regime state and reward based on action.

- `tab_Q_learning(...)`
  - Updates Q values using:
    - `Q[s,a] <- Q[s,a] + lr*(r + gamma*max_a' Q[s',a'] - Q[s,a])`

Output:
- `pi_star_Q`: greedy policy derived from learned Q-table

---

### 3) Deep Q-Learning (DQN)

DQN uses a neural network to approximate Q-values and uses experience replay plus a target network.

Key components:

- Model creation:
  - `create_model(in_size, hidden_size, out_size, lr)`
  - Network is a simple MLP with 2 hidden ReLU layers.

- State encoding:
  - `get_state_DQN(...)` encodes stock + regimes
  - `get_state_DQN2(...)` encodes stock + **recent reward changes** (scaled by 10)
  - The current code uses `get_state_DQN2`.

- Replay memory:
  - `memory = deque(maxlen=MAX_MEMORY)`
  - `remember(memory, state, action, reward, next_state)`

- Training step:
  - `train_step(...)` samples a random minibatch and performs gradient descent on an MSE temporal-difference target.

Output:
- `pi_star_DQN`: policy that picks argmax Q-value from the trained network

## Notes / Limitations

- This is a **toy model**:
  - Only two regimes per stock
  - Holds exactly one stock at a time (no portfolio weights)
  - Rewards are simple uniform noise around a mean
- Policy Iteration uses a full enumerated MDP:
  - State/action space grows as `N * 2^N`
  - Becomes infeasible quickly as `N` increases.
- DQN state encoding currently uses **recent sampled changes** (`get_state_DQN2`) rather than true regimes.
  - This can make learning harder/unstable depending on settings.
- DQN target network update condition:
  - Currently: `if i % 20 == 0: ...` inside the inner step loop.
  - Note: `i` is the episode index, so this triggers for *all steps* in episodes 0, 20, 40, ... (not “every 20 steps”).
